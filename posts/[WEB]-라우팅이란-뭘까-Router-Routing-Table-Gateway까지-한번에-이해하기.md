우리가 인터넷을 사용할 때는
브라우저에 주소를 입력하면 서버까지 데이터가 이동한다.

그런데 컴퓨터는 어떻게:

* 어디로 보내야 하는지
* 어떤 길로 가야 하는지
* 중간에 누구를 거쳐야 하는지

를 알까?

이걸 담당하는 핵심 개념이 바로:

* Routing
* Router
* Routing Table
* Gateway

이다.

오늘은 네트워크의 “길찾기 시스템”을 이해해보자.

---

# 1. Routing(라우팅)이란?

라우팅(Routing)은:

> 데이터를 목적지까지 어떤 경로로 보낼지 결정하는 과정

이다.

쉽게 말하면:

> “패킷 길찾기”

라고 생각하면 된다.

예를 들어:

```plaintext
내 컴퓨터 → 공유기 → 통신사 → 구글 서버
```

이렇게 중간중간 여러 장비를 거쳐서 이동한다.

이때:

* 어디로 보내야 하는지
* 다음 목적지가 누구인지

를 결정하는 게 라우팅이다.

---

# 2. 패킷(Packet)이란?

인터넷 데이터는 통째로 이동하지 않는다.

잘게 쪼개져서 이동한다.

이 조각 하나를:

```plaintext
Packet(패킷)
```

이라고 한다.

패킷 안에는:

```plaintext
출발지 IP
목적지 IP
데이터
```

등이 들어있다.

라우터는 이 목적지 IP를 보고:

> “어디로 보내야 하지?”

를 판단한다.

---

# 3. Router(라우터)란?

라우터는:

> 네트워크와 네트워크를 연결해주는 장비

이다.

대표적인 예:

* 집 공유기
* 회사 네트워크 장비
* 통신사 장비

전부 라우터 역할을 한다.

---

# 4. Router는 무슨 일을 할까?

라우터는 패킷을 받으면:

```plaintext
1. 목적지 IP 확인
2. 경로 확인
3. 다음 목적지로 전달
```

을 수행한다.

즉:

> “이 패킷 어디로 보내야 하지?”

를 계속 판단한다.

---

# 5. Routing Table(라우팅 테이블)

라우터는 머리로 기억하지 않는다.

대신:

```plaintext
Routing Table
```

이라는 표를 가지고 있다.

예시:

```plaintext
목적지           다음 위치
192.168.0.0      eth0
10.0.0.0         eth1
0.0.0.0          gateway
```

이 뜻은:

```plaintext
192.168 대역 → eth0으로 보내라
10 대역 → eth1으로 보내라
나머지 → gateway로 보내라
```

라는 의미다.

즉:

> 네트워크의 네비게이션 테이블

같은 개념이다.

---

# 6. 실제 흐름 예시

예를 들어:

```plaintext
내 PC: 192.168.0.10
구글 서버: 8.8.8.8
```

라고 해보자.

내 PC는:

```plaintext
8.8.8.8은 우리 집 네트워크가 아닌데?
```

라고 판단한다.

그러면:

```plaintext
기본 Gateway(공유기)로 보냄
```

공유기는 다시:

```plaintext
통신사 라우터로 전달
```

통신사 라우터는:

```plaintext
다음 인터넷 구간으로 전달
```

이걸 반복해서 최종 목적지까지 도달한다.

---

# 7. Gateway란?

Gateway는:

> 다른 네트워크로 나가는 출입구

이다.

보통 집에서는:

```plaintext
공유기 IP
```

가 Gateway다.

예시:

```plaintext
내 PC: 192.168.0.10
Gateway: 192.168.0.1
```

이 경우:

```plaintext
외부 인터넷 가고 싶으면
일단 192.168.0.1로 보내라
```

는 의미다.

---

# 8. 왜 Gateway가 필요할까?

컴퓨터는:

```plaintext
같은 네트워크
```

끼리는 직접 통신 가능하다.

예:

```plaintext
192.168.0.10
192.168.0.20
```

하지만:

```plaintext
8.8.8.8
```

같은 외부 네트워크는 직접 모른다.

그래서:

```plaintext
“밖으로 나가는 건 공유기한테 맡김”
```

이게 Gateway다.

---

# 9. Static Routing vs Dynamic Routing

라우팅 방식은 크게 두 가지다.

---

## 1) Static Routing

관리자가 직접 경로를 입력한다.

예:

```plaintext
10.0.0.0은 eth1으로 보내라
```

장점:

* 단순함
* 빠름

단점:

* 네트워크 커지면 관리 지옥

작은 네트워크에서 많이 사용한다.

---

## 2) Dynamic Routing

라우터끼리 자동으로 경로를 학습한다.

대표 프로토콜:

* RIP
* OSPF
* BGP

장점:

* 자동 경로 계산
* 장애 대응 가능

단점:

* 설정 복잡
* 리소스 사용

인터넷 규모에서는 거의 필수다.

---

# 10. 라우팅이 중요한 이유

라우팅이 없으면:

```plaintext
데이터가 어디로 가야 하는지 모름
```

즉 인터넷 자체가 동작하지 않는다.

실제로:

* AWS VPC Route Table
* Kubernetes 네트워크
* VPN
* BGP
* NAT Gateway
* 인터넷 통신

전부 라우팅 개념 위에서 동작한다.

---

# 11. DevOps/클라우드에서 왜 중요할까?

실무에서 진짜 많이 나온다.

예:

```plaintext
왜 서버 통신 안되지?
```

원인 보면:

* Route Table 문제
* Gateway 설정 문제
* Subnet 문제
* CIDR 충돌 문제

엄청 많다.

특히:

* AWS
* Kubernetes
* Docker
* VPN
* VPC

공부할 때 라우팅 이해도가 매우 중요하다.

---

# 12. 한줄 정리

```plaintext
Routing = 패킷 길찾기
Router = 길 안내 장비
Routing Table = 길찾기 지도
Gateway = 외부로 나가는 출입구
```

네트워크는 결국:

> “데이터를 어디로 어떻게 보낼 것인가”

를 해결하는 기술이다.
