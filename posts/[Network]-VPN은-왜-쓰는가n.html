[NETWORK] VPN은 왜 쓰는가\n

2026.05.11

회사 네트워크 공부를 하다 보면 꼭 나오는 게 있다.

VPN

그리고 같이 따라오는 단어들:

  • 터널링
  • 암호화
  • IPsec
  • Remote Access VPN
  • Site-to-Site VPN

처음엔 되게 어려워 보이는데, 사실 핵심은 하나다.

“인터넷 위에 안전한 전용 통로를 만드는 기술”

오늘은 이 흐름으로 이해해보자.

1. VPN은 왜 쓰는가?

먼저 생각해보자.

집에서 회사 서버에 접속한다고 해보자.

내 노트북
   ↓
인터넷
   ↓
회사 서버

문제는:

인터넷은 공용 공간

이라는 것이다.

즉:

  • 중간에서 훔쳐볼 수도 있고
  • 변조할 수도 있고
  • 누가 접속하는지도 알 수 있다

그래서 필요한 게:

VPN

이다.

2. VPN(Virtual Private Network)이란?

VPN은:

공용 인터넷 위에 마치 전용선처럼 안전한 네트워크를 만드는 기술

이다.

즉 실제로는 인터넷을 쓰는데:

"우리끼리만 연결된 사설망처럼 보이게"

만드는 것이다.

그래서 이름이:

Virtual(가상)
Private(사설)
Network(네트워크)

이다.

3. VPN의 핵심 2가지

VPN의 핵심은 거의 이것뿐이다.

1. 터널링
2. 암호화

4. 터널링(Tunneling)이란?

터널링은:

데이터를 다른 패킷 안에 넣어서 보내는 것

이다.

쉽게 말하면:

원래 데이터
↓
상자에 넣음
↓
인터넷으로 전달

느낌이다.

인터넷 중간 장비들은:

안에 뭐가 들었는지 잘 모름

상태가 된다.

그래서:

안전한 통로(터널)

라는 표현을 쓴다.

5. 암호화(Encryption)

근데 상자만 씌운다고 안전한 건 아니다.

중간에서 열어보면 끝이다.

그래서 VPN은 보통:

데이터 자체를 암호화

한다.

즉:

안녕하세요

가 아니라:

A8F2K9X...

같은 형태로 전달된다.

그래서 중간에서 패킷을 봐도:

무슨 내용인지 해석 불가

상태가 된다.

6. VPN 없이 접속하면?

예를 들어 공공 와이파이에서:

카페 Wi-Fi

를 사용한다고 해보자.

VPN 없이 접속하면:

내 트래픽
→ 인터넷에 그대로 노출 가능

하다.

특히 예전에는:

  • 스니핑
  • 세션 탈취
  • 패킷 감청

문제가 꽤 많았다.

그래서 회사들은 보통:

사내망 접근 시 VPN 필수

를 걸어둔다.

7. Remote Access VPN

이게 가장 흔한 형태다.

구조

집
↓
인터넷
↓
회사 VPN 서버
↓
사내망

즉 외부 직원이:

회사 내부망에 안전하게 접속

하는 것이다.

재택근무할 때 많이 쓴다.

8. Site-to-Site VPN

이번엔 사람 말고:

회사 ↔ 회사

를 연결한다고 해보자.

예시:

서울 본사
↔
부산 지사

둘을 안전하게 연결해야 한다.

이때 쓰는 게:

Site-to-Site VPN

이다.

구조

지사 라우터
↓
인터넷
↓
본사 라우터

근데 VPN 터널로 연결되어 있어서:

마치 하나의 내부망처럼 통신

가능하다.

9. Remote Access vs Site-to-Site 차이

종류 목적
Remote Access VPN 사용자 접속
Site-to-Site VPN 네트워크끼리 연결

10. IPsec이란?

VPN 이야기하면 거의 같이 나오는 게:

IPsec

이다.

11. IPsec(IP Security)

IPsec은:

IP 패킷 자체를 보호하는 보안 프로토콜 모음

이다.

쉽게 말하면:

VPN 만들 때 사용하는 대표 보안 기술

이다.

12. IPsec이 하는 일

IPsec은 크게:

1. 암호화
2. 무결성 검증
3. 인증

을 담당한다.

1) 암호화

데이터 내용을 숨김.

2) 무결성 검증

중간에서 데이터가 변조됐는지 확인.

3) 인증

상대가 진짜 회사 장비인지 확인.

13. IPsec의 핵심 프로토콜

실무에서 자주 보이는 건:

AH
ESP

이다.

AH(Authentication Header)

무결성과 인증 담당.

근데:

암호화는 안 함

그래서 요즘은 잘 안 쓴다.

ESP(Encapsulating Security Payload)

이게 핵심이다.

ESP는:

  • 암호화
  • 인증
  • 무결성

전부 지원한다.

실무 VPN 대부분은:

ESP 기반

이라고 생각해도 된다.

14. VPN이 클라우드에서도 중요한 이유

AWS에서도 VPN 많이 쓴다.

예시:

회사 IDC
↔
AWS VPC

를 연결할 때:

Site-to-Site VPN

을 사용한다.

즉:

온프레미스 ↔ 클라우드

를 안전하게 연결하는 것이다.

15. VPC와 VPN 연결 흐름

예를 들어:

회사 내부망:
192.168.0.0/16

AWS VPC:
10.0.0.0/16

를 연결하면:

회사 서버
↔
AWS EC2

가 내부망처럼 통신 가능해진다.

16. 정리

VPN

인터넷 위에 안전한 사설망 생성

터널링

데이터를 감싸서 전달

암호화

데이터 내용을 숨김

Remote Access VPN

사용자가 회사망 접속

Site-to-Site VPN

회사 네트워크끼리 연결

IPsec

VPN에서 많이 사용하는 보안 기술

ESP

암호화 + 인증 + 무결성 지원

한 줄 핵심

VPN은 공용 인터넷 위에
암호화된 전용 통로를 만드는 기술이다.
