네트워크를 공부하다 보면 꼭 나오는 게 있다.

* Subnet
* CIDR
* `/24`
* Subnet Mask
* VPC

처음 보면 다 비슷하게 생겨서 헷갈린다.

근데 사실 이건 전부:

> “IP 주소를 어떻게 나눠서 관리할 것인가”

를 설명하는 개념들이다.

오늘은 이 흐름을 한 번에 정리해보자.

---

# 1. IP 주소란?

IP 주소는 네트워크 상의 컴퓨터 주소다.

예시:

```text
192.168.0.10
```

사람으로 치면:

```text
서울시 강남구 테헤란로 123
```

같은 개념이다.

컴퓨터끼리 통신하려면:

```text
"누구한테 보낼 건데?"
```

를 알아야 하니까 주소(IP)가 필요하다.

---

# 2. 근데 IP를 왜 나누는가?

만약 회사 전체가 IP를 그냥 막 쓴다고 해보자.

```text
192.168.0.1
192.168.0.2
192.168.0.3
...
```

수천 대가 한 네트워크에 붙어있으면:

* 브로드캐스트 증가
* 관리 어려움
* 충돌 위험
* 보안 문제

가 생긴다.

그래서 등장한 게:

```text
Subnet
```

이다.

---

# 3. Subnet(서브넷)이란?

Subnet은:

> 하나의 큰 네트워크를 작은 네트워크로 나누는 것

이다.

예를 들어:

```text
192.168.0.0 ~ 192.168.255.255
```

이 큰 네트워크를:

```text
192.168.1.x
192.168.2.x
192.168.3.x
```

처럼 여러 구역으로 나누는 것.

아파트 단지에서:

```text
101동
102동
103동
```

나누는 느낌과 비슷하다.

---

# 4. Subnet Mask란?

여기서 등장하는 게:

```text
Subnet Mask
```

이다.

얘는:

> 어디까지가 네트워크 주소인지 구분하는 마스크

다.

예시:

```text
IP:
192.168.0.10

Subnet Mask:
255.255.255.0
```

이 뜻은:

```text
192.168.0
```

여기까지가 네트워크 영역이라는 뜻이다.

그리고:

```text
10
```

이 부분은 컴퓨터(host)를 의미한다.

즉:

```text
192.168.0.xxx
```

는 같은 네트워크라는 뜻이다.

---

# 5. 255.255.255.0은 왜 저런 숫자인가?

사실 컴퓨터 내부에서는:

```text
11111111.11111111.11111111.00000000
```

이렇게 본다.

1은:

```text
네트워크 영역
```

0은:

```text
호스트 영역
```

이다.

즉:

```text
11111111 = 255
```

라서:

```text
255.255.255.0
```

가 되는 것이다.

---

# 6. CIDR이란?

근데 매번:

```text
255.255.255.0
```

이렇게 쓰기 귀찮다.

그래서 등장한 게:

```text
CIDR
(Classless Inter-Domain Routing)
```

이다.

CIDR은:

> 네트워크 비트 개수를 간단하게 표기하는 방식

이다.

---

# 7. `/24` 는 무슨 뜻인가?

예시:

```text
192.168.0.0/24
```

이 뜻은:

```text
앞의 24비트가 네트워크 영역
```

이라는 뜻이다.

즉:

```text
11111111.11111111.11111111.00000000
```

와 동일하다.

그래서:

```text
/24 = 255.255.255.0
```

이다.

---

# 8. `/16`, `/24`, `/32` 차이

## `/16`

```text
255.255.0.0
```

가능한 IP 수:

```text
약 65536개
```

네트워크 범위 큼.

예:

```text
192.168.x.x
```

---

## `/24`

```text
255.255.255.0
```

가능한 IP 수:

```text
256개
```

가장 자주 보인다.

예:

```text
192.168.0.x
```

---

## `/32`

```text
255.255.255.255
```

호스트 영역이 없음.

즉:

```text
IP 하나만 지정
```

하는 의미다.

보통:

* 특정 서버 허용
* 방화벽 정책
* 라우팅 대상

같은 데서 많이 쓴다.

예:

```text
10.0.0.5/32
```

→ 딱 한 IP만 의미.

---

# 9. 왜 CIDR이 중요한가?

현대 클라우드는 거의 CIDR 기반이다.

AWS VPC 생성할 때도:

```text
10.0.0.0/16
```

같이 설정한다.

이 뜻은:

```text
이 VPC 전체가 사용할 네트워크 범위
```

라는 뜻이다.

그리고 그 안에서:

```text
10.0.1.0/24
10.0.2.0/24
```

처럼 Subnet을 나눈다.

---

# 10. VPC와 Subnet 연결

AWS에서 보면:

```text
VPC
 ├─ Public Subnet
 ├─ Private Subnet
 └─ DB Subnet
```

이런 구조가 나온다.

예시:

```text
VPC:
10.0.0.0/16
```

안에서:

```text
Public:
10.0.1.0/24

Private:
10.0.2.0/24

DB:
10.0.3.0/24
```

처럼 쪼개는 것이다.

즉:

```text
큰 네트워크(VPC)
→ 작은 네트워크(Subnet)
```

구조다.

---

# 11. 왜 Public / Private Subnet을 나누는가?

예를 들면:

## Public Subnet

인터넷 접근 가능

```text
- ALB
- Bastion
- Web Server
```

---

## Private Subnet

인터넷 직접 접근 불가

```text
- API Server
- DB
- Redis
```

보안을 위해 내부망으로 숨기는 것이다.

실무에서는 거의 항상:

```text
Public / Private 분리
```

를 한다.

---

# 12. 정리

## IP 주소

컴퓨터 주소

---

## Subnet

큰 네트워크를 작은 네트워크로 분리

---

## Subnet Mask

네트워크 영역 구분

```text
255.255.255.0
```

---

## CIDR

Subnet Mask를 짧게 표현

```text
/24
```

---

## `/24`

```text
255.255.255.0
```

약 256개 IP

---

## `/16`

큰 네트워크

---

## `/32`

IP 하나만 의미

---

## VPC/Subnet

```text
VPC = 큰 네트워크
Subnet = 내부 구역 분리
```

---

# 한 줄 핵심

```text
CIDR은 네트워크를 얼마나 크게 나눌지 표현하는 방식이고,
Subnet은 실제로 네트워크를 나눈 결과다.
```
