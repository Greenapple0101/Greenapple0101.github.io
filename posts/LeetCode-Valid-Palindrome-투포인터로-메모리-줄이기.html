[LeetCode] Valid Palindrome — 투포인터로 메모리 줄이기

2026.05.11 Velog 원문

처음에는 문자열에서 알파벳과 숫자만 뽑아서 리스트에 저장한 뒤 비교할 수 있다.

class Solution:
    def isPalindrome(self, s: str) -> bool:
        arr = []

        for ch in s:
            if ch.isalnum():
                arr.append(ch.lower())

        for i in range(len(arr) // 2):
            if arr[i] != arr[len(arr) - i - 1]:
                return False

        return True

이 풀이는 직관적이다. 하지만 arr라는 새 리스트를 만들기 때문에 메모리를 추가로 사용한다.

시간복잡도: O(n)
공간복잡도: O(n)

그래서 메모리를 줄이려면 투포인터를 쓴다.

투포인터 아이디어

왼쪽 끝과 오른쪽 끝에서 동시에 출발한다.

left  →           ← right

문자나 숫자가 아니면 건너뛴다.

not s[left].isalnum()

문자나 숫자면 소문자로 바꿔서 비교한다.

s[left].lower() != s[right].lower()

다르면 팰린드롬이 아니므로 False.

끝까지 통과하면 True.

코드

class Solution:
    def isPalindrome(self, s: str) -> bool:
        left = 0
        right = len(s) - 1

        while left < right:
            while left < right and not s[left].isalnum():
                left += 1

            while left < right and not s[right].isalnum():
                right -= 1

            if s[left].lower() != s[right].lower():
                return False

            left += 1
            right -= 1

        return True

앞의 while문들은 특수문자를 건너뛰는 용도고,

while left < right and not s[left].isalnum(): left += 1

마지막 left += 1, right -= 1은 정상 문자 비교 후 다음 문자로 넘어가는 용도야.

흐름 예시

s = "A man, a plan, a canal: Panama"

비교 대상만 남기면:

amanaplanacanalpanama

앞뒤가 계속 같으므로 True.

반대로:

s = "0P"

비교하면:

0 != p

그래서 False.

왜 공간복잡도가 O(1)인가?

리스트를 새로 만들지 않는다.

문자열 안에서 인덱스만 움직인다.

left
right

이 두 변수만 사용한다.

그래서:

시간복잡도: O(n)
공간복잡도: O(1)

정리

리스트 방식은 이해하기 쉽다.

필터링 → 저장 → 앞뒤 비교

투포인터 방식은 메모리를 아낀다.

왼쪽/오른쪽에서 직접 비교

코딩테스트에서는 둘 다 통과 가능하지만, 메모리까지 신경 쓰는 풀이로는 투포인터가 더 깔끔하다.
